3 — Filter System (useFilters.js)

A singleton composable — all views share the same reactive state. Changing a filter immediately triggers watchers in every active view and re-fetches data via api.js.

6 — Request Data Flow

Trace of a user selecting Warehouse = Tokyo on the Inventory page.

1

User Action — FilterBar.vue

User picks "Tokyo" from the warehouse dropdown. <select v-model="selectedLocation"> updates the shared composable ref.

2

Composable Update — useFilters.js

selectedLocation.value = "Tokyo". All components sharing this composable react immediately because it's a module-level singleton.

3

Watcher Fires — Inventory.vue

watch([selectedLocation, selectedCategory], () => loadInventory()) triggers. Calls getCurrentFilters() → { warehouse: "Tokyo", category: "all" }.

4

HTTP Request — api.js

axios.get('http://localhost:8001/api/inventory?warehouse=Tokyo'). The "all" sentinel values are excluded from the query string.

5

Backend Filter — main.py

apply_filters(inventory_items, warehouse="Tokyo") iterates the in-memory list and returns items where item['warehouse'] == "Tokyo". Pydantic serializes the response.

6

Computed Property — Inventory.vue

items.value is set to the API response. filteredItems (computed) applies any local search query and sorts by stock status: Low Stock → Adequate → In Stock.

7

Template Render — Inventory.vue

v-for="item in filteredItems" :key="item.id" re-renders the table rows. Vue's virtual DOM diffs and applies only the changed rows.
